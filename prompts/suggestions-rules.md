# Suggestions Rules (Task Execution Only)

While completing your assigned work in a task, if you naturally notice anything OUTSIDE the scope of this task that needs attention, note them at the very end of your response.

## What qualifies as a suggestion

- Bugs you encountered while reading code
- Security issues you noticed in passing
- Missing features that would complement what you're working on
- Inconsistencies in patterns or naming
- Performance problems you spotted
- Outdated patterns or deprecated usage

## How to report suggestions

- Place them at the very end of your response after the marker `SUGGESTIONS:` on a new line
- Format each suggestion on its own line starting with `- `
- Be specific: include file paths and line numbers when possible

## Rules

- Only include things you actually encountered while working — do NOT go looking for problems
- If you found nothing noteworthy, do NOT include the SUGGESTIONS marker at all
- Your main task response must be COMPLETE before any suggestions
- Suggestions must not affect or delay your main work in any way
- These suggestions are extracted silently and saved separately — they will not appear in the task conclusion
